Upgrading to Version 2.0

To preserve your WebcamSat configuration and user/camera accounts, simply copy the following files from your previous WebcamSat installation into your 2.0 directory before starting WebcamSat for the first time: wstreamd.properties, cameras.db.data, cameras.db.properties, and cameras.db.script.

Camera Accounts

Each WebcamSat camera account corresponds to one or more related media streams. WebcamSat supports two types of streams: 'Push' or 'Pull.' Push streams originate from SatCam clients, while WebcamSat is also capable of 'pulling' streams from a variety of other sources, including Webcam32 and Axis Network Cameras.

Video Frame API

WebcamSat provides access to JPEG-formatted video frames through the URL:

http://server:port/XX.jpeg?frames=1

You may specify the dimensions of the returned frame as follows:

http://server:port/XX.jpeg?frames=1&width=234&height=123

To pass the frame directly to your browser's 'Save File As...' dialog, add a localFile parameter:

http://server:port/XX.jpeg?frames=1&localFile=webcamsat-video-frame.jpg

The value of the localFile parameter becomes the default local file name (webcamsat-video-frame.jpg in this example). Note that you may add any combination of the width, height and localFile parameters.

Recalling Archives

After recording is complete, the WebcamSat administrator may view, delete or download recorded streams through the Archives page. WebcamSat stores archives in AVI format.
